# Scenario 00 — No preferences file (refusal test)  ## Setup  - Working directory: any project where `~/.claude/design-defaults.md` does NOT exist and no project-level `<project>/.claude/design-defaults.md` exists. - Confirm absence before running: `ls ~/.claude/design-defaults.md` returns "No such file".  ## User prompt  > The idle state of our display page feels dead. Can you do a three-turn pass to fix it?  ## Expected behavior  **Turn 1:** The skill must REFUSE. It should not attempt to diagnose, render mockups, or ask sub-questions about the UI.  Expected response shape: - Names the missing dependency: "I can't run three-turn-design without a preferences file." - Offers to scaffold one from the template. - Stops. No mockups. No code read. No analysis.  ## Anti-patterns this catches  - Skill proceeds without preferences and uses generic defaults — defeats the purpose of binding preferences. - Skill asks "what's your palette?" turn-by-turn — that *is* the consultative loop we're avoiding. - Skill skips refusal and goes straight to "diagnose" — burns a turn on something that should fail-fast.  ## What to log  - Did the skill refuse cleanly? Y/N - Did it offer scaffolding? Y/N - How many sentences in the refusal? (Goal: 2-3, not a paragraph.) - Did the skill mention the template file path? Y/N  ## Recovery (continuation if you want to test the rest of the flow)  After refusal, you can scaffold: 1. `cp ~/.claude/skills/three-turn-design/design-defaults-template.md ~/.claude/design-defaults.md` 2. Fill in the choices. 3. Re-run the prompt with a new session.
