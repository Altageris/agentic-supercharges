Glossary: Technical Terms

Terms used in the Accessibility Analysis Reports, simplified and explained.

Anonymization

The process of removing identifying signals from content (names, numbers, paths) to make it portable and privacy-preserving

Structural Equivalence

Two systems are structurally equivalent if they have the same logical flow and control patterns, even if variable names differ

Commit Hash

A unique identifier for a specific version of code in a git repository (e.g., a8ba1f6)

Error Propagation

How an error manifests and spreads through a system from its root cause to observable symptoms

Normalization

Standardizing terminology and structure to make content readable and portable across systems

Redaction

Replacing sensitive or identifying information with generic placeholders (e.g., , )

Metadata

Structured data that describes other data, used for automated processing and discovery

Manifest

A registry or index file that lists and describes artifacts, their versions, and relationships
